std::string objsearch::sysutil::trimPath ( std::string  inPath,
int  nToTrim,
bool  fromFront 
)

Trim the given path back by nToTrim elements. A path element is either a file or a directory.

The path /home/user/data/file.txt has four elements. Three are directories and one is a filename. The elements are trimmed from the end of the path by default. Calling the function on this path with nToTrim=1 returns /home/user/data. The same with fromFront=true gives user/data/file.txt. Calling with nToTrim=-1 gives file.txt, and with fromFront=true as well, gives /home.

Parameters
pathThe path to trim
nToTrimThe number of elements to trim from the path. If the argument given is negative, the behaviour changes so that instead of trimming the elements from the starting point, it trims them from the end point, leaving the number of elements specified by this parameter.
fromFrontWhen true, trim the elements from the front of the path instead of the back.
Returns
A path with the given number of elements trimmed. If nToTrim exceeds the total number of elements, then you will be left with either a filename if fromFront is true, or an empty string otherwise. Trimming /home/user/data/file.txt by 2 elements from the back would give you /home/user. Notice that there is no trailing slash. Going from the front the return would be data/file.txt.
